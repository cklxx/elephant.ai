# 客户数据接入 Checklist（企业微信 AI 跟进服务）

> 目标：签约后 24 小时内完成数据接入，确保 Day 1 可启动 7 天 MVP。

## A. 企业微信授权与系统接入
- [ ] 提供企业微信管理员账号（仅用于授权，不留存明文密码）
- [ ] 完成企业微信应用授权（会话存档/客户联系/标签管理权限）
- [ ] 配置回调地址与 token（用于消息事件与标签变更同步）
- [ ] 确认 API 调用白名单/IP 白名单
- [ ] 建立测试账号（至少 2 个客服/销售账号）

## B. 标签体系对接
- [ ] 提供现有客户标签清单（字段名、含义、取值范围）
- [ ] 映射三类核心标签：
  - [ ] 生命周期标签（新客/活跃/沉默/流失风险）
  - [ ] 交易标签（已购、到期时间、续费状态）
  - [ ] 意向标签（高/中/低）
- [ ] 定义标签更新规则（自动更新条件 + 人工兜底规则）
- [ ] 完成标签字段冲突检查（重名、空值、脏数据）

## C. 历史数据导出与校验
- [ ] 导出近 90 天会话记录（至少包含客户ID、时间、消息内容）
- [ ] 导出近 180 天交易/续费记录（订单号、金额、时间、状态）
- [ ] 导出客户基础资料（来源渠道、首次接触时间、负责人）
- [ ] 完成数据脱敏约定（手机号/身份证等敏感字段）
- [ ] 完成抽样校验（随机 30 条记录，准确率 >= 98%）

## D. 合规与质检初始化
- [ ] 提供敏感词清单（监管词、禁用承诺词、投诉高风险词）
- [ ] 确认频控策略（单客户触达频次、时段限制）
- [ ] 确认人工接管阈值（高风险会话升级条件）
- [ ] 确认留痕审计周期（日报/周报）

## E. 启动前验收（Go/No-Go）
- [ ] 三个场景（售后回访/续费提醒/沉默激活）均有可用样本数据
- [ ] API 连通性通过（成功率 >= 99%）
- [ ] 测试账号收发消息正常
- [ ] 触发规则干跑（Dry-run）通过
- [ ] 双方确认 Day 1 Kickoff 时间

## 交付责任矩阵（RACI）
- 客户负责人（A）：授权与业务规则确认
- 客户技术接口人（R）：数据导出与字段说明
- 服务方项目经理（A）：整体进度与验收
- 服务方实施工程师（R）：接入、映射、联调
- 服务方质检（C）：合规词库与抽检策略

## 预计时长
- 标准接入：4-8 小时
- 含历史数据清洗：8-16 小时
