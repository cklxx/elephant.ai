# 客户接入 24 小时快速启动 Checklist

## 0-2 小时：商务与权限准备
- [ ] 确认已收取意向金（>=999 元）并创建客户项目编号
- [ ] 拉通客户对接群（业务负责人 + 运营 + 技术）
- [ ] 收集企业微信基础信息（CorpID、应用可见范围、管理员）
- [ ] 签署数据授权与合规确认（会话数据用途、留存周期、审计边界）

## 2-6 小时：技术接入
- [ ] 配置环境变量（`WECOM_CORP_ID` / `WECOM_CORP_SECRET` / `WECOM_AGENT_ID`）
- [ ] 运行 `wecom_data_connector.py` 验证基础连通性
- [ ] 完成客户标签映射（行业标签、意向标签、生命周期标签）
- [ ] 验证外部联系人拉取与标签同步日志

## 6-12 小时：SOP 部署
- [ ] 运行 `sop_deployer.py` 部署 3 套 SOP：售后回访 / 续费提醒 / 沉默激活
- [ ] 与业务方确认触发条件与频次策略
- [ ] 完成敏感词与频控规则加载
- [ ] 抽样验证 5-10 条自动流程执行记录

## 12-24 小时：看板与上线验收
- [ ] 运行 `daily_dashboard_generator.py` 生成日报样例
- [ ] 校验核心指标口径：触达率、回复率、转化率
- [ ] 输出首日基线报告（用于后续对比）
- [ ] 设定 72 小时 A/B 复盘节奏（话术/时段/触达策略）

## 上线验收标准
- [ ] 首响链路可用，失败可重试
- [ ] 三套 SOP 均可触发并有日志
- [ ] 日报可自动生成并可归档
- [ ] 合规规则（敏感词/频控）可审计

## 兼容性说明（wecom-delivery-package-builder）
- 当前目录结构与交付包生成器保持一致：`scripts/`、`configs/`、`docs/`
- 环境变量命名遵循既有规范，可直接纳入已有打包流程
- SOP 与日报输出采用标准 JSON/日志格式，便于后续接入统一构建与归档
