# 会议纪要（可分发与可执行）

## When to use this skill
- 需要把会议/访谈/同步的零散笔记整理成结构化纪要。  
- 需要明确决策、行动项、风险、待澄清问题并分配 Owner/截止时间。

## 必备输入
- 会议基本信息：主题、日期与时区、参会人及角色、会议目标。  
- 原始记录：要点、讨论片段、截屏/链接。  
- 希望的分发范围与语气（正式/简洁）。  
- 是否允许公开未定结论或敏感信息的处理要求。

## 工作流
1. **梳理上下文**：确认会议目的与预期输出（决策/方案对齐/风险沟通）。  
2. **抽取核心块**：`决策`、`行动项`、`风险/阻塞`、`待澄清问题`、`重要讨论摘要`。  
3. **结构化输出**：推荐顺序如下：  
   - 元信息（时间、参会人、目标）  
   - TL;DR（三到五条结论或进展）  
   - 决策（含影响范围/生效时间）  
   - 行动项（Owner + Deadline + 成功标准）  
   - 风险与缓解（风险描述、影响、当前缓解或需要的支持）  
   - 待澄清问题（问题 + 需要谁答复 + 时限）  
   - 讨论摘要（按议题归档，保留关键信息与数据口径）  
4. **验证**：检查是否遗漏 owner、时间或度量标准；术语一致；链接可点击。  
5. **分发说明**：如有受众差异，可附“外发版”与“内部全量版”的差异说明。

## 输出格式示例
- 以 Markdown 段落+列表呈现；行动项可用表格或列表：`- [ ] Owner — 任务（截止：日期，标准：...）`。  
- 对未决事项，用 `TBD` 标明，并指明预期答复时间。

## 最终检查清单
- 每条行动项都有 Owner 与截止时间。  
- 决策写清“做什么 + 为什么 + 影响范围/生效时间”。  
- 风险项含风险级别与缓解计划；待澄清问题明确责任人。  
- 若有外部分享需求，确认敏感信息已处理（脱敏/移除）。
