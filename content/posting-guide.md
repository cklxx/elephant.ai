# 各平台发布操作指南（首批试点招募）

## 目标与口径
- 核心目标：引导潜在客户私信/加微信 `tcckl1`，拿到首批 3 家试点沟通。
- 统一卖点：
  - 7天试点
  - 5分钟首响SLA
  - 48小时二次唤醒
  - 无效退款（写清适用条件：按约完成数据接入与SOP执行）
- 统一CTA：
  - 私信关键词：`试点`
  - 微信：`tcckl1`

---

## 1) 小红书发布步骤（偏痛点+案例感）
1. 选择「图文笔记」或「短视频+文案」。
2. 标题建议：
   - `客服回复慢、复购低？我们做了个7天企微试点（无效退款）`
3. 正文使用“痛点-结果-机制-CTA”结构（已在 social-recruitment-posts.md 提供）。
4. 标签建议（6-10个）：
   - #企业微信 #私域运营 #AI客服 #客服效率 #复购增长 #运营提效 #SOP
5. 发布后 15 分钟内，主动回复前 5 条评论，优先引导私信。

最佳发布时间（本地时间）
- 工作日：11:30-13:30，19:30-22:30
- 周末：10:00-12:00，20:00-23:00

评论区维护话术
- 询问价格：
  - `先看你们当前链路和客资规模，我这边先给你做7天试点方案，合适再定价。私信我「试点」就行。`
- 质疑AI效果：
  - `我们不做“全自动忽悠”，规则是 AI分层+SOP，人机协同，重点看SLA和回复率提升。`
- 观望用户：
  - `首批只收3家，主要是保证交付密度。你要不要先占个评估名额？`

## 2) 即刻发布步骤（偏技术/效率工具）
1. 发到「产品汪」「独立开发」「AI应用」「运营增长」等相关圈子。
2. 开头先给“可执行框架”：AI自动分层 + SOP + 质检看板。
3. 明确限制条件：仅招募 3 家，便于形成稀缺感。
4. 结尾放 CTA：私信关键词 + 微信号。

最佳发布时间
- 工作日：08:30-10:00，12:00-13:00，18:30-21:30
- 周末：10:00-12:00，19:00-22:00

评论区维护话术
- 技术细节追问：
  - `核心不是模型炫技，而是流程闭环：分层→首响→二次唤醒→预约确认→日报复盘。`
- 问“是不是割韭菜”：
  - `我们按结果说话，7天看SLA和回复率，不达标按约退款。`
- 要案例：
  - `可先给你看脱敏版日报结构，重点是指标口径可审计。`

## 3) 知乎发布步骤（长文深度）
1. 用文章体裁发布《2026年企微私域的破局点：AI轻托管模式》。
2. 目录建议：
   - 背景（为什么传统私域失效）
   - 方法（AI轻托管三段式）
   - 指标（SLA/回复率/预约率）
   - 风险（合规、归因、交付）

3. 配 1 张流程图（文字也可）：
   - 线索分层 → 5分钟首响 → 48h唤醒 → 预约确认 → 日报复盘
4. 文末固定引导：
   - `首批试点咨询：微信 tcckl1（备注：试点）`

最佳发布时间
- 工作日：12:00-14:00，20:00-23:00
- 周末：10:00-12:00，20:00-23:00

评论区维护话术

## 4) Boss直聘 / 脉脉发布步骤（面向决策者）
1. 用“业务合作/项目合作”口径发，不用“招聘客服”口吻。
2. 结构：

   - 方案（7天试点）
   - 结果承诺（无效退款）
   - 联系方式（微信）
3. 文案要短，避免技术术语堆砌。

最佳发布时间
- 工作日：09:00-11:00，14:00-16:00，20:00-22:00
- 周末：10:00-12:00，20:00-22:00

评论区/私信维护话术
- 决策者问投入产出：

## 发布后运营SOP（统一）
- T+0（发布后30分钟内）
  - 回复首批评论，优先引导私信“试点”。
- T+1天
  - 补1条跟进动态：展示方法论或脱敏数据结构。
- T+3天
  - 复盘四项指标：曝光、互动、私信数、加微数。
- T+7天
  - 做一期“试点进展”复盘贴，继续引流。

## 指标看板（最低追踪）
- 每平台：曝光量、互动率、私信数、加微数、有效商机数
- 总漏斗：
  - 内容曝光 -> 私信咨询 -> 加微信 -> 约诊断会 -> 支付试点

## 失败兜底策略
- 若互动低：
  - 改标题，强化结果承诺（7天试点+无效退款）
- 若私信高但加微低：
  - 评论区增加“先发试点清单再加微”降低心理门槛
- 若加微高但成交低：
  - 首次沟通改为“15分钟漏斗诊断”，先给可执行建议再报价
