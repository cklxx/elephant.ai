## Vision

elephant.ai today is a competent reactive agent with proactive features bolted on.
The target state is an **autonomous operating system for one person's professional
attention** — proactive-with-reactive-fallback, not the other way around.

### Key Shift

- You never check Lark first. It checks you.
- Cross-tool signal fusion is real-time, not scheduled polling.
- The agent learns your decision patterns and gradually automates low-risk decisions.
- Subagent parallelism is invisible.
- Memory compounds over time — after 6 months, the agent knows your rhythm.

### Platonic Ideal

A brilliant, tireless chief of staff. Morning brief arrives: "2 items need your call."
You reply with decisions. Before your 1:1, a prep brief arrives unprompted. You never
open Jira, GitHub, or search Lark history. The agent brought the signal to you,
pre-synthesized, with clear action handles.

## Accepted Expansions (6)

### 1. Signal Graph Architecture

Replace scheduled polling (10-min blocker scan, hourly milestones, Monday pulse) with
a real-time event-driven signal graph. Events from Lark, Git, Calendar flow into an
attention scorer that decides what to surface and when.

- **Package:** `internal/app/signals/{source,scorer,router}/`
- **Feature flag:** `signal_graph.enabled`
- **Effort:** human ~2 weeks / CC ~2-4 hours
- **Unlocks:** Morning Brief, Phase 2 multi-source fusion (Jira/Git as plugins)

### 2. Decision Learning Engine

Observe user decisions, extract patterns, build confidence model, gradually automate
low-risk decisions with audit trail and undo window.

- **Package:** `internal/app/decision/{engine,pattern,confidence}/`
- **Feature flag:** `decision_engine.auto_act` (start observe-only)
- **Effort:** human ~3 weeks / CC ~3-5 hours
- **Depends on:** Memory Distillation (#5)
- **Safety:** checksum validation, integrity scan, 90% confidence floor, 24h undo

### 3. Unified Session Model + Surface Handoff

Merge Lark/Web/CLI session handling into one surface-agnostic store. Start a task in
CLI, see progress in Lark, review in Web.

- **Package:** `internal/infra/session/unified/`
- **Migration strategy:** Dual-write (old + new stores) for 1-2 weeks, validate parity, then cut over
- **Effort:** human ~2 weeks / CC ~2-3 hours
- **Unlocks:** CLI Runtime P0 (builds on correct session contract)

### 4. Morning Auto-Brief + "What Did I Miss?"

First message of day triggers a synthesized brief: blockers needing judgment, decisions
to make, wins to celebrate. On-demand "what did I miss?" command.

- **Consumer of:** Signal Graph (#1)
- **UX:** Lark card with action buttons (Approve/Skip/Edit/Remind)
- **Effort:** human ~3 days / CC ~30 min
- **Empty state:** "All clear — nothing needs your attention"

### 5. Memory Self-Distillation Pipeline

Daily: raw conversations -> extracted decisions/facts.
Weekly: daily extracts -> patterns.
Monthly: patterns -> personality model.

- **Builds on:** Memory D5 restructuring (already P1)
- **Package:** `internal/infra/memory/distillation/`
- **Effort:** human ~2 weeks / CC ~2-3 hours
- **Supercharges:** Decision Engine (#2)

### 6. Weekly Self-Report + Operator Observability

Agent sends weekly meta-report: tasks handled, accuracy rate, corrections received,
decisions auto-handled vs escalated, cost spent, what it learned.

- **Depends on:** Decision Engine (#2) for decision metrics
- **Effort:** human ~2 days / CC ~20 min
- **Purpose:** Trust feedback loop for autonomous operation
