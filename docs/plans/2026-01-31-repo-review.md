# 仓库广泛代码审查计划

Updated: 2026-01-31

## 目标
- 广泛扫描性能与设计问题，给出带定位的风险清单。

## 范围
- `internal/`
- `cmd/`
- `web/`
- 关键运行时路径（事件流、工具调用、存储、调度）

## 计划
1. ✅ 收集上下文与既有性能/架构约束（docs/memory + guides）。
2. ✅ 快速扫描热点与风险模式（I/O 限制、日志、事件流、调度、缓存/保留策略）。
3. ✅ 深读高风险区域并记录具体定位与影响。
4. ✅ 运行全量 lint + tests，记录结果（lint 有 1 处 staticcheck 失败）。
5. ✅ 输出审查结论与建议。
